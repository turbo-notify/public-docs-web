O aviso de digitação permite que sua aplicação sinalize ao destinatário que uma mensagem está sendo preparada, simulando uma experiência de conversa mais natural.

## Aviso Automático

O Turbo Notify gerencia automaticamente o aviso de digitação antes do envio de mensagens. Quando uma mensagem é enviada, o indicador de digitação é ativado por um breve período antes da entrega, proporcionando uma experiência mais humana sem necessidade de configuração adicional.

Use os endpoints abaixo apenas quando precisar de controle manual sobre o indicador de digitação.

## Início da Digitação

Para sinalizar que uma mensagem está sendo preparada, envie uma requisição com `value` igual a `true`.

## Aviso com número extra

Para escolher o remetente do aviso de digitação, informe o campo `from` com um [número extra](/extra-numbers/add/) cadastrado e ativo na sua conta.

```bash title="typing.sh"
curl -X POST \
  https://api.turbonotify.com/typing-indicator \
  -H "Authorization: Bearer SUA_CHAVE" \
  -H "Content-Type: application/json" \
  -d '{
    "from": {"alias": "vendas-sp"},
    "to": {"number": "+5511999999999"},
    "value": true
  }'
```

Também é possível usar o alias definido no cadastro do número extra para identificar o remetente de forma mais prática.

Caso o número ou alias informado no campo `from` não estiver cadastrado como número extra ou não estiver ativo, a operação será rejeitada.

Saiba mais sobre os números extra em [Números Extra](/extra-numbers/add/).

## Eventos de webhook

A conclusão da operação chega via webhook com os seguintes eventos:

| Evento            | Descrição                                             |
| ----------------- | ----------------------------------------------------- |
| `typing.started`  | O aviso de digitação foi ativado com sucesso.         |
| `typing.stopped`  | O aviso de digitação foi interrompido com sucesso.    |
| `typing.failed`   | Ocorreu um erro ao processar o aviso de digitação.    |

## Erros possíveis

| Status | Código de erro          | Descrição                                          |
| :----: | ----------------------- | -------------------------------------------------- |
| `400`  | `invalid_json`          | O JSON da requisição está malformado ou inválido.  |
| `402`  | `not_available_in_plan` | O recurso não está disponível no plano contratado. |
| `403`  | `not_authorized`        | A chave de acesso está inválida ou ausente.        |
| `422`  | `contact_not_found`     | O número informado em `to` não foi encontrado.     |
| `422`  | `sender_not_authorized` | O campo `from` não está ativo na conta.            |
| `500`  | `internal_error`        | Erro interno inesperado no servidor.               |
| `503`  | `whatsapp_unavailable`  | O WhatsApp está temporariamente fora do ar.        |
