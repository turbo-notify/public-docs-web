O aviso de digitacao permite que sua aplicacao sinalize ao destinatario que uma mensagem esta sendo preparada, simulando uma experiencia de conversa mais natural.

## Aviso Automatico

O Turbo Notify gerencia automaticamente o aviso de digitacao antes do envio de mensagens. Quando uma mensagem e enviada, o indicador de digitacao e ativado por um breve periodo antes da entrega, proporcionando uma experiencia mais humana sem necessidade de configuracao adicional.

Use os endpoints abaixo apenas quando precisar de controle manual sobre o indicador de digitacao.

## Inicio da Digitacao

Para sinalizar que uma mensagem esta sendo preparada, envie uma requisicao com `value` igual a `true`.

## Aviso com numero extra

Para escolher o remetente do aviso de digitacao, informe o campo `from` com um [numero extra](/extra-numbers/add/) cadastrado e ativo na sua conta.

```bash title="typing.sh"
curl -X POST \
  https://api.turbonotify.com/typing-indicator \
  -H "Authorization: Bearer SUA_CHAVE" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "+5511888888888",
    "to": "+5511999999999",
    "value": true
  }'
```

Tambem e possivel usar o alias definido no cadastro do numero extra para identificar o remetente de forma mais pratica.

Caso o numero ou alias informado no campo `from` nao estiver cadastrado como numero extra ou nao estiver ativo, a operacao sera rejeitada.

Saiba mais sobre os numeros extra em [Numeros Extra](/extra-numbers/add/).

## Eventos de webhook

A conclusao da operacao chega via webhook com os seguintes eventos:

| Evento            | Descricao                                             |
| ----------------- | ----------------------------------------------------- |
| `typing.started`  | O aviso de digitacao foi ativado com sucesso.         |
| `typing.stopped`  | O aviso de digitacao foi interrompido com sucesso.    |
| `typing.failed`   | Ocorreu um erro ao processar o aviso de digitacao.    |

## Erros possiveis

| Status | Codigo de erro          | Descricao                                          |
| :----: | ----------------------- | -------------------------------------------------- |
| `400`  | `invalid_json`          | O JSON da requisicao esta malformado ou invalido.  |
| `402`  | `not_available_in_plan` | O recurso nao esta disponivel no plano contratado. |
| `403`  | `not_authorized`        | A chave de acesso esta invalida ou ausente.        |
| `422`  | `contact_not_found`     | O numero informado em `to` nao foi encontrado.     |
| `422`  | `sender_not_authorized` | O campo `from` nao esta ativo na conta.            |
| `500`  | `internal_error`        | Erro interno inesperado no servidor.               |
| `503`  | `whatsapp_unavailable`  | O WhatsApp esta temporariamente fora do ar.        |
