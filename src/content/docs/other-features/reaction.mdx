O recurso de reacao permite que sua aplicacao envie ou remova reacoes em mensagens especificas, tornando as interacoes mais expressivas.

## Adicionar uma Reacao

Este servico e utilizado para adicionar uma reacao a uma mensagem enviada.

## Reacao com numero extra

Para escolher o remetente da reacao, informe o campo `from` com um [numero extra](/extra-numbers/add/) cadastrado e ativo na sua conta.

```bash title="reaction.sh"
curl -X POST \
  https://api.turbonotify.com/reactions \
  -H "Authorization: Bearer SUA_CHAVE" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "+5511888888888",
    "to": "+5511999999999",
    "message_id": "msg_abc123",
    "reaction": "🎉"
  }'
```

Tambem e possivel usar o alias definido no cadastro do numero extra para identificar o remetente de forma mais pratica.

Caso o numero ou alias informado no campo `from` nao estiver cadastrado como numero extra ou nao estiver ativo, a operacao sera rejeitada.

Saiba mais sobre os numeros extra em [Numeros Extra](/extra-numbers/add/).

## Erros possiveis

| Status | Codigo de erro          | Descricao                                          |
| :----: | ----------------------- | -------------------------------------------------- |
| `400`  | `invalid_json`          | O JSON da requisicao esta malformado ou invalido.  |
| `402`  | `not_available_in_plan` | O recurso nao esta disponivel no plano contratado. |
| `403`  | `not_authorized`        | A chave de acesso esta invalida ou ausente.        |
| `422`  | `message_not_found`     | O `message_id` nao foi encontrado.                 |
| `422`  | `sender_not_authorized` | O campo `from` nao esta ativo na conta.            |
| `500`  | `internal_error`        | Erro interno inesperado no servidor.               |
| `503`  | `whatsapp_unavailable`  | O WhatsApp esta temporariamente fora do ar.        |

Para entender mais sobre o erro `message_not_found`, consulte a documentacao sobre [Retencao](/messages/retention/).
