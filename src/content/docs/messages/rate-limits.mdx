O Turbo Notify aplica limites de uso para garantir estabilidade, segurança e qualidade do serviço para todos os clientes. Os limites variam de acordo com o plano contratado e o comportamento de envio.

## Tipos de limite

| Limite              | Descrição                                                                     |
| ------------------- | ----------------------------------------------------------------------------- |
| Mensagens por minuto | Quantidade máxima de mensagens que podem ser enviadas por minuto.             |
| Mensagens por hora   | Quantidade máxima de mensagens que podem ser enviadas por hora.               |
| Mensagens por dia    | Quantidade máxima de mensagens que podem ser enviadas em um período de 24h.   |
| Requisições por minuto | Quantidade máxima de chamadas à API por minuto, independente do endpoint.  |

Os valores exatos de cada limite dependem do plano contratado. Consulte os detalhes do seu plano no dashboard ou entre em contato com o suporte.

## Números extra

Cada número extra cadastrado na conta possui seus próprios limites de envio, independentes do número principal. Os limites de cada número extra seguem as mesmas regras do plano contratado.

Isso permite distribuir o volume de envio entre vários números, aumentando a capacidade total de envio da sua conta.

## Controles internos

Além dos limites do plano, o Turbo Notify aplica controles internos para proteger a infraestrutura e evitar abusos. Esses controles podem restringir temporariamente o envio em situações como:

- Picos anormais de tráfego vindos de uma única conta.
- Padrões de envio que possam comprometer a estabilidade do serviço.
- Alta taxa de falhas ou rejeições consecutivas.

Esses controles são transparentes e, na maioria dos casos, não impactam o uso normal da plataforma.

## Risco de bloqueio

O WhatsApp possui suas próprias regras e limites de uso. Mesmo que você esteja dentro dos limites do Turbo Notify, o envio excessivo, não solicitado ou considerado spam pelos destinatários pode resultar em bloqueio do seu número pelo WhatsApp.

Para evitar problemas, siga as recomendações de uso responsável descritas em [Evite o Abuso](/general/abuse/) e as <ExternalLink href="https://www.whatsapp.com/legal/business-policy/">políticas do WhatsApp</ExternalLink>.

## Resposta ao exceder o limite

Quando o limite é atingido, a API retorna o status `429` com o cabeçalho `Retry-After` indicando quantos segundos você deve aguardar antes de tentar novamente.

<ApiResponse
  status={429}
  headers={{ 'Retry-After': '30' }}
  body={{
    error: {
      code: 'rate_limit_exceeded',
      message: 'Limite de envio excedido. Tente novamente em 30 segundos.',
    },
  }}
/>

## Erros relacionados

| Status | Código de erro          | Descrição                                                  |
| :----: | ----------------------- | ---------------------------------------------------------- |
| `429`  | `rate_limit_exceeded`   | Limite de envio do plano ultrapassado.                     |
| `429`  | `too_many_requests`     | Limite de requisições por minuto ultrapassado.             |
| `503`  | `service_unavailable`   | Serviço temporariamente indisponível por sobrecarga.       |

## Boas práticas

- Respeite o cabeçalho `Retry-After` ao receber um status `429`.
- Distribua o envio ao longo do tempo em vez de enviar tudo de uma vez.
- Utilize números extra para aumentar a capacidade de envio.
- Monitore os códigos de resposta da API para identificar rapidamente quando um limite é atingido.
- Configure [webhooks](/messages/webhook/) para acompanhar o status das mensagens sem precisar consultar a API repetidamente.
