Recupere o conteúdo completo, o status de entrega e a referência de mídia de qualquer mensagem enviada ou recebida na sua conta, usando o `message_id` como identificador.

**Escopo obrigatório:** `messages:read`

## Campos da resposta

| Campo | Tipo | Descrição |
| ----- | ---- | --------- |
| `message_id` | string | Identificador público da mensagem (`msg_` + 32 hex) |
| `direction` | string | `outbound` (enviada) ou `inbound` (recebida) |
| `from` | string | Número do remetente |
| `to` | string | Número do destinatário |
| `content_type` | string | `text`, `image`, `audio`, `video`, `document`, `sticker`, `location`, `contact` ou `reaction` |
| `status` | object | `{state, at}` — estado atual de entrega |
| `preview` | string | Permanente — primeiros 50 caracteres do conteúdo, sempre presente |
| `body` | object\|null | Conteúdo estruturado completo; `null` quando expirado pela política de retenção |
| `body_expired` | boolean | `true` quando o corpo foi removido pelo ciclo de retenção |
| `media` | object\|null | Bloco de mídia; `null` para tipos não-mídia |
| `created_at` | string | Timestamp ISO 8601 de criação |
| `updated_at` | string | Timestamp ISO 8601 da última atualização de status |

### Bloco de mídia

Presente quando `content_type` é `image`, `audio`, `video`, `document` ou `sticker`:

| Campo | Tipo | Descrição |
| ----- | ---- | --------- |
| `storage` | string | `whatsapp_direct` (URL efêmera do WhatsApp) ou `tenant_s3` (URL pré-assinada do seu bucket) |
| `url` | string\|null | URL de download; `null` quando `media_expired: true` |
| `url_expires_at` | string | Timestamp de expiração da URL |
| `mime_type` | string | Tipo MIME do arquivo (ex: `image/jpeg`, `application/pdf`) |
| `size_bytes` | integer | Tamanho do arquivo em bytes |
| `media_expired` | boolean | `true` quando o TTL de ~24h do WhatsApp expirou e a URL não está mais disponível |

### Sobre `body_expired`

O campo `body` contém o conteúdo estruturado completo da mensagem. Após o período de retenção (padrão: 30 dias), o corpo é removido e `body_expired` passa a ser `true`. O campo `preview` — primeiros 50 caracteres — é permanente e sempre estará presente.

### Sobre `media_expired`

URLs de mídia do WhatsApp expiram em aproximadamente 24 horas. Quando `media_expired: true`, a `url` retorna `null`. Se você precisar de acesso permanente à mídia recebida, configure a <DashboardLink path="/settings/integrations/storage">integração de Object Storage</DashboardLink> no Dashboard para que o Turbo Notify copie automaticamente a mídia para o seu bucket.

## Erros possíveis

| Status | Código de erro | Descrição |
| :----: | -------------- | --------- |
| `400` | `invalid_message_id` | O identificador não corresponde ao formato `msg_` + 32 hex. |
| `403` | `not_authorized` | A chave de acesso está inválida, ausente ou sem o escopo `messages:read`. |
| `404` | `not_found` | Mensagem não encontrada ou pertence a outro tenant. |
| `429` | `rate_limit_exceeded` | Limite de requisições ultrapassado. |
| `500` | `internal_error` | Erro interno inesperado no servidor. |
