Envie mensagens pelo Turbo Notify de forma simples, segura e em conformidade com as <ExternalLink href="https://www.whatsapp.com/legal/business-policy/">políticas do WhatsApp</ExternalLink>.

O corpo de `POST /v1/messages` usa um campo `content` discriminado por
`content_type`, suportando texto, localização, contato, calendário e
mídia (`image`, `video`, `audio`, `document`, `sticker` — veja
[Envio de Mídia](/messages/send-media/)).

Para citar (responder) uma mensagem existente — qualquer `content_type` — adicione o campo opcional `reply_to_message_id` no body do request com o id público da mensagem citada (`msg_{hex32}`). O webhook resultante é `message.replied` (em vez de `message.sent`); veja [Webhooks](/messages/webhook/#exemplo--messagereplied-inboundoutbound).

## Envio básico

O envio mais simples exige apenas o número do destinatário e o texto da mensagem. O Turbo Notify usará automaticamente o número principal da sua conta como remetente.

    curl -X POST \
      https://api.turbonotify.com/v1/messages \
      -H "Authorization: Bearer SUA_CHAVE" \
      -H "Content-Type: application/json" \
      -d '{
        "to": {"number": "+5511999999999"},

Também é possível usar o alias definido no cadastro do número extra para identificar o remetente de forma mais prática.

Caso o número ou alias informado no campo `from` não estiver cadastrado como número extra ou não estiver ativo, o envio será rejeitado.

Saiba mais sobre os números extra em [Números Extra](/extra-numbers/add/).

## Erros possíveis

Veja os principais erros que podem ocorrer no envio de mensagens e como interpretá-los:

| Status | Código de erro          | Descrição                                         |
| :----: | ----------------------- | ------------------------------------------------- |
| `400`  | `invalid_json`          | O JSON da requisição está malformado ou inválido. |
| `403`  | `not_authorized`        | A chave de acesso está inválida ou ausente.       |
| `422`  | `sender_not_authorized` | O campo `from` não está ativo na conta.           |
| `429`  | `rate_limit_exceeded`   | Limite de envio do plano ultrapassado.            |
| `500`  | `internal_error`        | Erro interno inesperado no servidor.              |
| `503`  | `whatsapp_unavailable`  | O WhatsApp está temporariamente fora do ar.       |

## Status da mensagem

O status informa o andamento do processamento da mensagem. Para detalhes completos sobre cada status, consulte a documentação [Status](/messages/status/).

<ApiResponse
  status={202}
  body={{
    message_id: 'msg_abc123',
    operation_id: 'op_msg_001',
    status: { state: 'pending', at: '2025-07-16T10:00:00Z' },
  }}
/>

O retorno final do processamento chega via webhook com os eventos `message.sent`, `message.delivered`, `message.read` ou `message.failed`.

## Segurança e Privacidade

Toda comunicação entre seu sistema e a API do Turbo Notify ocorre exclusivamente por HTTPS, assegurando que os dados estejam sempre criptografados durante o tráfego.

Para saber por quanto tempo as mensagens ficam disponíveis para consulta e operações, veja a documentação de [Retenção](/messages/retention/).

## Conteúdo rico — Localização, Contato e Calendário

A partir de 2026-04-27 o `POST /v1/messages` aceita três novos
`content_type` além de `text`. Cada variante tem um esquema próprio
e é despachada para o WhatsApp como o tipo nativo correspondente
(`locationMessage`, `contactMessage`, `eventMessage`).

:::

### Como aparece no destinatário

Cada tipo aparece no WhatsApp do destinatário com a UI nativa:
**localização** com mapa e botão "Ver no mapa", **contato** com card
salvável, **calendário** com botão "Adicionar ao calendário".

Para visualizar exatamente como o evento `message.sent` correspondente
chega no seu webhook, abra o [Webhook Inspector](https://webhook.turbonotify.com/)
— o painel "Enriched" renderiza esses três tipos com cards dedicados.
