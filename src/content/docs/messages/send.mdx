Envie mensagens pelo Turbo Notify de forma simples, segura e em conformidade com as <ExternalLink href="https://www.whatsapp.com/legal/business-policy/">politicas do WhatsApp</ExternalLink>.

## Envio basico

O envio mais simples exige apenas o numero do destinatario e o texto da mensagem. O Turbo Notify usara automaticamente o numero principal da sua conta como remetente.

Tambem e possivel usar o alias definido no cadastro do numero extra para identificar o remetente de forma mais pratica.

Caso o numero ou alias informado no campo `from` nao estiver cadastrado como numero extra ou nao estiver ativo, o envio sera rejeitado.

Saiba mais sobre os numeros extra em [Numeros Extra](/extra-numbers/add/).

## Erros possiveis

Veja os principais erros que podem ocorrer no envio de mensagens e como interpreta-los:

| Status | Codigo de erro          | Descricao                                         |
| :----: | ----------------------- | ------------------------------------------------- |
| `400`  | `invalid_json`          | O JSON da requisicao esta malformado ou invalido. |
| `403`  | `not_authorized`        | A chave de acesso esta invalida ou ausente.       |
| `422`  | `sender_not_authorized` | O campo `from` nao esta ativo na conta.           |
| `429`  | `rate_limit_exceeded`   | Limite de envio do plano ultrapassado.            |
| `500`  | `internal_error`        | Erro interno inesperado no servidor.              |
| `503`  | `whatsapp_unavailable`  | O WhatsApp esta temporariamente fora do ar.       |

## Status da mensagem

O status informa o andamento do processamento da mensagem. Para detalhes completos sobre cada status, consulte a documentacao [Status](/messages/status/).

<ApiResponse
  status={202}
  body={{
    message_id: 'msg_abc123',
    operation_id: 'op_msg_001',
    status: { state: 'pending', at: '2025-07-16T10:00:00Z' },
  }}
/>

O retorno final do processamento chega via webhook com os eventos `message.sent`, `message.delivered`, `message.read` ou `message.failed`.

## Seguranca e Privacidade

Toda comunicacao entre seu sistema e a API do Turbo Notify ocorre exclusivamente por HTTPS, assegurando que os dados estejam sempre criptografados durante o trafego.

Para saber por quanto tempo as mensagens ficam disponiveis para consulta e operacoes, veja a documentacao de [Retencao](/messages/retention/).
