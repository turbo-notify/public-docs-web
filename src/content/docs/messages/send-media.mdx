Mídia (imagens, áudios, vídeos, documentos e figurinhas) é enviada
pelo mesmo endpoint `POST /v1/messages` usado para texto. Cada tipo é
uma variante do `content` discriminado por `content_type`.

:::caution[Implementação em progresso]
O contrato HTTP é final, mas o upload outbound de mídia para o
WhatsApp está em finalização (gateway neonize). Requests para os 5
tipos de mídia abaixo são aceitos pela API e geram `message_id`, mas
o worker pode retornar `message.send.failed` com `whatsapp_send_failed`
até o release que completa o gateway. Acompanhe o webhook
`message.sent` para confirmar o envio.
:::

## Tipos de mídia suportados

| `content_type` | Formatos comuns | Tamanho máximo | `caption`? | `filename`? |
| -------------- | --------------- | :------------: | :--------: | :---------: |
| `image` | JPEG, PNG, WebP | 16 MB | ✓ | — |
| `video` | MP4 | 100 MB | ✓ | — |
| `audio` | MP3, OGG, M4A | 16 MB | — | — |
| `document` | PDF, DOCX, XLSX, etc. | 100 MB | ✓ | ✓ |
| `sticker` | WebP animado | 500 KB | — | — |

`caption` é descartado server-side em `audio` e `sticker` — o WhatsApp
ignora o campo nesses tipos.

## Modalidade — URL pública

Forneça uma URL HTTPS acessível publicamente. O Turbo Notify busca o
arquivo antes de entregar ao WhatsApp.

## Esquema de campos (todos os tipos)

| Campo | Tipo | Obrigatório | Notas |
|---|---|---|---|
| `content.content_type` | enum | sim | `image` / `video` / `audio` / `document` / `sticker` |
| `content.url` | URL | sim | apenas `http://` / `https://` |
| `content.mime_type` | string | não | recomendado; o worker pode inferir do `Content-Type` da resposta |
| `content.caption` | string | parcial | aceito em `image`, `video`, `document`; **descartado** em `audio` e `sticker` |
| `content.filename` | string | não | apenas em `document`; aparece como nome do anexo no chat |

## Resposta

```json title="response.json"
{
  "message_id": "msg_a1b2c3d4e5f60718293a4b5c6d7e8f90",
  "status": {
    "state": "pending",
    "at": "2026-04-27T10:00:05.123456+00:00",
    "reason": null
  }
}
```

A confirmação real chega via webhook `message.sent`. Em caso de falha
(URL inacessível, formato não suportado, tamanho excedido) você recebe
`message.send.failed` com `error_code` específico.

## Webhook outbound

Quando o envio é confirmado pelo WhatsApp, o webhook `message.sent`
carrega `content_type` e `preview` (mesmo shape de `message.received`):

```json title="webhook-message-sent.json"
{
  "id": "evt_abc123def456789012345678",
  "type": "message.sent",
  "request_id": "req_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4",
  "timestamp": "2026-04-27T10:00:08Z",
  "data": {
    "message_id": "msg_a1b2c3d4e5f60718293a4b5c6d7e8f90",
    "from": { "number": "5511999990001", "alias": "main" },
    "to":   { "number": "5511999999999" },
    "content_type": "image",
    "preview": "Foto do produto",
    "sent_at": "2026-04-27T10:00:08Z"
  }
}
```

Para baixar o arquivo de uma mídia recebida, consulte
[`GET /v1/messages/{id}`](/messages/get/) — o body inclui `media.url`
com URL pré-assinada (BYO Object Storage) ou URL direta da WhatsApp.
