O Turbo Notify suporta envio de mensagens de mídia: imagens, áudios, vídeos, documentos e figurinhas. Existem duas formas de fornecer o arquivo: por **URL pública** ou por **chave de objeto S3** (requer integração BYO S3 configurada).

## Tipos de mídia suportados

| `content_type` | Formatos comuns | Tamanho máximo |
| -------------- | --------------- | :------------: |
| `image` | JPEG, PNG, WebP | 16 MB |
| `audio` | MP3, OGG, M4A | 16 MB |
| `video` | MP4 | 100 MB |
| `document` | PDF, DOCX, XLSX, etc. | 100 MB |
| `sticker` | WebP animado | 500 KB |

## Modalidade 1 — URL pública

Forneça uma URL HTTPS acessível publicamente. O Turbo Notify busca o arquivo no servidor antes de entregar ao WhatsApp.

### Restrição SSRF

A URL fornecida deve apontar para um servidor público. As seguintes origens são bloqueadas por segurança:

- IPs privados: `10.x.x.x`, `172.16–31.x.x`, `192.168.x.x`, `127.x.x.x`
- Metadata AWS: `169.254.169.254`
- Metadata GCP: `metadata.google.internal`
- Qualquer esquema diferente de `https://`

Tentativas de usar URLs bloqueadas retornam `422 media_url_blocked`.

## Campos da requisição

| Campo | Tipo | Obrigatório | Descrição |
| ----- | ---- | :---------: | --------- |
| `to` | string | Sim | Número do destinatário com DDI |
| `content_type` | string | Sim | `image`, `audio`, `video`, `document` ou `sticker` |
| `media.url` | string | Condicional | URL HTTPS pública. Exclusivo com `s3_object_key` |
| `media.s3_object_key` | string | Condicional | Chave do objeto no bucket S3 configurado. Exclusivo com `url` |
| `media.caption` | string | Não | Legenda exibida abaixo da mídia. Máximo 1024 caracteres. Não suportado em `audio` e `sticker` |
| `from` | string | Não | Telefone ou alias de um número extra |

Exatamente um dos campos `media.url` ou `media.s3_object_key` deve ser fornecido.

## Resposta de sucesso

<ApiResponse
  status={202}
  body={{
    message_id: 'msg_01993c5a1b24d3e9f01a2b3c4d5e6f70',
    status: { state: 'pending', at: '2026-04-21T10:00:00Z' },
  }}
/>

O resultado final é entregue de forma assíncrona via webhook com os eventos `message.sent` ou `message.send.failed`.

## Erros possíveis

| Status | Código de erro | Descrição |
| :----: | -------------- | --------- |
| `400` | `invalid_json` | O JSON da requisição está malformado ou inválido. |
| `403` | `not_authorized` | A chave de acesso está inválida ou ausente. |
| `422` | `media_url_blocked` | A URL aponta para um host bloqueado (proteção SSRF). |
| `413` | `media_too_large` | O arquivo ultrapassa o limite para este tipo de conteúdo. |
| `422` | `byo_s3_not_configured` | `media.s3_object_key` foi fornecido mas nenhuma integração S3 está configurada. |
| `422` | `sender_not_authorized` | O campo `from` não está ativo na conta. |
| `429` | `rate_limit_exceeded` | Limite de envio do plano ultrapassado. |
| `500` | `internal_error` | Erro interno inesperado no servidor. |
| `503` | `whatsapp_unavailable` | O WhatsApp está temporariamente fora do ar. |
