---
title: Retenção das Mensagens
description: Como funciona o ciclo de vida das mensagens no Turbo Notify.
---

O Turbo Notify adota uma política de retenção curta para garantir a privacidade dos dados e manter a infraestrutura leve e eficiente.

## Armazenamento e Privacidade

O Turbo Notify não armazena o conteúdo das mensagens de forma permanente. O texto enviado é utilizado apenas para processar o envio e, após a conclusão, os dados são descartados automaticamente conforme o prazo de retenção.

Essa abordagem prioriza a privacidade dos seus usuários e está alinhada com as melhores práticas de proteção de dados.

## Prazo de retenção

O status e os metadados da mensagem ficam disponíveis para consulta durante um período limitado. Após esse prazo, a mensagem deixa de existir no sistema.

| Período após o envio | Consulta de status | Resposta da API |
| -------------------- | :----------------: | :-------------: |
| Até 8 horas          | Disponível         | `200`           |
| Entre 8 e 24 horas   | Expirado           | `410`           |
| Após 24 horas        | Removido           | `404`           |

### Até 8 horas

A mensagem e seus metadados estão totalmente disponíveis. Você pode consultar o status, verificar se foi entregue, lida ou se houve falha.

### Entre 8 e 24 horas

A mensagem entra em período de expiração. A API retorna `410 Gone`, indicando que o recurso existiu mas não está mais disponível. Nenhum dado é retornado.

### Após 24 horas

A mensagem é completamente removida do sistema. A API retorna `404 Not Found`, como se a mensagem nunca tivesse existido.

## Implicações práticas

- Consulte o status da mensagem o mais cedo possível após o envio.
- Configure um [webhook](/messages/webhook/) para receber atualizações em tempo real e não depender exclusivamente da consulta manual.
- Armazene no seu sistema os dados que forem relevantes para o seu negócio, como o `message_id` e o status final.
- Operações que dependem da mensagem original, como [responder](/messages/send/) (via campo `reply_to_message_id`), também estão sujeitas ao prazo de retenção.

## Consulta de status

Para verificar o status de uma mensagem dentro do prazo de retenção, veja a documentação de [Status](/messages/status/).
