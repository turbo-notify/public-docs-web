Apos enviar uma mensagem, voce pode consultar o status dela a qualquer momento pela API. O status reflete o andamento do processamento e entrega no WhatsApp.

## Tabela de status

| Status      | Descricao                                                                 |
| ----------- | ------------------------------------------------------------------------- |
| `pending`   | A mensagem foi recebida pelo Turbo Notify e aguarda processamento.        |
| `sent`      | A mensagem foi enviada ao WhatsApp com sucesso.                           |
| `delivered` | A mensagem foi entregue ao dispositivo do destinatario.                   |
| `read`      | A mensagem foi lida pelo destinatario.                                    |
| `failed`    | O envio falhou. Consulte o campo `reason` para detalhes sobre a falha.    |

## Consultar status

Para consultar o status de uma mensagem, faca uma requisicao `GET` informando o `message_id` retornado no envio.

## Motivos de falha

Quando o status e `failed`, o campo `reason` indica o motivo. Veja os valores possiveis:

| Motivo                  | Descricao                                                        |
| ----------------------- | ---------------------------------------------------------------- |
| `whatsapp_unavailable`  | O WhatsApp esta temporariamente fora do ar ou indisponivel.      |
| `blocked`               | O destinatario bloqueou o remetente ou o numero foi bloqueado.   |

## Erros possiveis

| Status | Codigo de erro   | Descricao                                            |
| :----: | ---------------- | ---------------------------------------------------- |
| `403`  | `not_authorized` | A chave de acesso esta invalida ou ausente.          |
| `404`  | `not_found`      | A mensagem nao foi encontrada ou ja expirou.         |
| `500`  | `internal_error` | Erro interno inesperado no servidor.                 |
| `503`  | `service_unavailable` | O servico esta temporariamente indisponivel.    |

## Retencao

As mensagens ficam disponiveis para consulta por um periodo limitado. Apos esse prazo, o status nao podera mais ser consultado. Para detalhes, veja a documentacao de [Retencao](/messages/retention/).

## Atualizacoes em tempo real

Para receber atualizacoes automaticas sobre o status das mensagens sem precisar consultar a API repetidamente, configure um webhook. Veja como em [Respostas e Webhook](/messages/webhook/).
