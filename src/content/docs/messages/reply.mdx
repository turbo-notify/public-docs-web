Responda mensagens recebidas criando um encadeamento visual no WhatsApp. A resposta aparece vinculada a mensagem original, facilitando o acompanhamento da conversa pelo destinatario.

## Enviar resposta

Para responder uma mensagem, faca uma requisicao `POST` informando o `message_id` da mensagem original na URL e o texto da resposta no corpo.

<Tabs>
  <TabItem label="cURL">
    ```bash title="turbo-notify.sh"
    curl -X POST \
      https://api.turbonotify.com/messages/msg_abc123/reply \
      -H "Authorization: Bearer SUA_CHAVE" \
      -H "Content-Type: application/json" \
      -d '{
        "text": "Obrigado pela sua mensagem!"
      }'
    ```
  </TabItem>
  <TabItem label="Node">
    ```js title="turbo-notify.js"
    const response = await fetch(
      'https://api.turbonotify.com/messages/msg_abc123/reply',

        method: 'POST',
        headers: {
          'Authorization': 'Bearer SUA_CHAVE',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          text: 'Obrigado pela sua mensagem!'
        })
      }
    );

    const result = await response.json();
    console.log(result);
    ```
  </TabItem>
  <TabItem label="Python">
    ```python title="turbo_notify.py"

        "text": "Obrigado pela sua mensagem!"
    ```

  </TabItem>
  <TabItem label="Go">
    ```go title="turbo_notify.go"

        req.Header.Set("Authorization", "Bearer SUA_CHAVE")
        req.Header.Set("Content-Type", "application/json")
        resp, _ := http.DefaultClient.Do(req)
        defer resp.Body.Close()
        body, _ := io.ReadAll(resp.Body)
        fmt.Println(string(body))
    }
    ```

    ```java title="TurboNotify.java"
                {

                """;

        }
    }
    ```

## Idempotencia

Para evitar respostas duplicadas em caso de falhas de rede ou retentativas, utilize o cabecalho `Idempotency-Key` com um valor unico por requisicao.

## Acompanhamento

A resposta segue o mesmo fluxo de status de uma mensagem comum. Acompanhe o andamento pela API ou via webhook. Veja mais em [Status](/messages/status/).

## Erros possiveis

| Status | Codigo de erro            | Descricao                                                       |
| :----: | ------------------------- | --------------------------------------------------------------- |
| `400`  | `invalid_json`            | O JSON da requisicao esta malformado ou invalido.               |
| `403`  | `not_authorized`          | A chave de acesso esta invalida ou ausente.                     |
| `422`  | `message_not_found`       | A mensagem original nao foi encontrada ou ja expirou.           |
| `422`  | `sender_not_authorized`   | O campo `from` nao esta ativo na conta.                         |
| `422`  | `reply_not_allowed`       | A mensagem original nao permite resposta.                       |
| `500`  | `internal_error`          | Erro interno inesperado no servidor.                            |
| `503`  | `whatsapp_unavailable`    | O WhatsApp esta temporariamente fora do ar.                     |
