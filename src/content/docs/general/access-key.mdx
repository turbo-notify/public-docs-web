Para utilizar a API do Turbo Notify, é indispensável possuir uma chave de acesso válida.

O gerenciamento dessas chaves é feito de forma simples e segura pelo dashboard, permitindo total controle sobre quem pode interagir com sua conta e seus números.

## Inclusão

Novas chaves podem ser geradas diretamente no dashboard, acessando o menu <DashboardLink path="/access-keys">Chaves de Acesso</DashboardLink>. É possível criar até 100 chaves por conta, facilitando integrações com diferentes sistemas ou equipes.

O valor da chave é exibido apenas uma vez, no momento da criação. Por segurança, copie e armazene imediatamente, pois não será possível visualizar novamente.

## Cabeçalho

Toda requisição à API deve conter o header `Authorization` com o valor da chave de acesso. Sem esse cabeçalho, nenhuma chamada será autorizada.

```http
Authorization: Bearer SUA_CHAVE
```

## Revogação

Uma chave de acesso pode ser temporariamente revogada ou permanentemente excluída, conforme a necessidade de controle sobre o acesso à sua conta.

Para revogação temporária, basta desativar a chave pelo dashboard. Enquanto estiver desativada, qualquer tentativa de uso será bloqueada pela API, mas você pode reativá-la a qualquer momento, restabelecendo o acesso.

Se desejar remover o acesso de forma definitiva, exclua a chave pelo dashboard. Após a exclusão, não será mais possível recuperar ou reativar a chave, e todas as requisições feitas com ela serão permanentemente bloqueadas.

## Permissão

Todas as chaves criadas têm permissão para enviar mensagens tanto para o número principal da conta quanto para os números extra associados. Não é possível limitar o acesso de uma chave a um número específico; todas têm acesso irrestrito aos números vinculados à conta.

Caso precise restringir o acesso a determinados números, é necessário criar contas separadas para cada número desejado. Essa limitação existe para evitar uso indevido do Turbo Notify.
